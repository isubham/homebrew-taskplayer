Lists are artists, tasks are songs — albums sit in between, grouping related tasks inside one list. Four ways to surface that grouping, all built on the existing look. Same example throughout: the Website Relaunch list/artist, with two albums (Backend, Marketing Site) plus a couple of one-off tasks that don't belong to any album ("singles").

1 · Album sections in the track list

Closest to how Spotify's own artist page groups a discography: the flat task table gets split into sections, one per album, each with its own small header — album tile (reuses the list's own color/emoji language, just one size down), name, track count + total time, and a play button that starts the album's first unfinished task. Tasks with no album fall into a plain "Singles" section at the bottom. Reordering / progress bars / everything else about a row is untouched.

2 · Inline album tag + filter chips

Smallest change: the table stays exactly as flat as it is today, but each row that belongs to an album gets a small colored pill next to its name (same visual language as the existing "deep"/"shallow" depth tag). A row of filter chips above the table — All, Singles, then one per album — lets you narrow the view down to just one album without leaving the list. Good fit if reordering tasks freely across album boundaries matters more than a clean grouped view.

3 · Sidebar drill-down (artist → album)

Treats an album like a sub-list: the artist row in the sidebar gets a chevron and expands in place to show its albums (plus an implicit "Singles" row for ungrouped tasks), each with its own track count. Clicking an album swaps the main view to just that album's tracks, with a small "Website Relaunch › Backend" breadcrumb up top instead of the plain list name. Closest to how the app already treats Lists vs. Recent as separate nav destinations — an album becomes a third kind of destination.

4 · Album gallery as a tab strip

A horizontally-scrolling row of album "covers" sits between the list header and the task table — visually the most Spotify-like of the four, closest to an artist page's album shelf. Each card is really just a filter tab: click one to narrow the table below to that album, click "All" to go back to the full list. No sidebar changes, no separate page — the whole thing lives inside the existing list view.
