One horizontal strip per day, sitting where the day's sessions already live on the Sessions page — right under the "Today · 2h 27m" header. Each session becomes a colored block positioned at its real start time and sized to its real duration (same list colors as the dots in the row list below it), so a glance at the strip alone tells you the shape of the day. All four variants use the same sample day: two migration sessions in the morning, a standup, an afternoon block, a short docs session, and a session still running now.

1 · Full 24h, midnight to midnight

The simplest version: the bar is always the full day, so "9am" and "9pm" always land in the same place no matter what happened that day. Ticks at 12a/6a/12p/6p/12a. Honest and predictable, at the cost of the overnight two-thirds of the bar usually sitting empty.

3 · Waking window only (6am–midnight)

Drops the overnight hours nobody's tracking anything in, so the same 640px now covers an 18h window instead of 24 — every session gets ~33% more room, short ones stop needing a minimum-width floor as often. Trade-off: the scale is day-to-day consistent but no longer literally "midnight to midnight," which is a small honesty cost for a lot of readability.
