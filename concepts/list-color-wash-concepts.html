1 · Full-bleed wash

The simplest fix: swap the header's fixed grey gradient for one keyed off listItem.color — strong at the top, fading to black by the toolbar. Costs nothing structurally (same .main rule, just a variable instead of a hardcoded grey), and it's the single biggest reason today's header reads the same for every list.

3 · Color as the whole accent, not just the header

The boldest option: the list's color replaces green everywhere on that list's page — the big play button, the active/playing row, the toolbar pill hover. Every list gets its own accent instead of all of them sharing the same Spotify green; green stays exactly where it is today in Settings and the global player bar, so "focus mode" still always reads as green from across the room.

Which to build

1 is the closest to real Spotify and the easiest lift — one CSS rule, reading a value the app already stores. 2 is safer if a full color wash feels like too much for something meant to stay in the background all day. 3 is a bigger, more opinionated change (touches the play button, row highlight, and toolbar everywhere) but makes the emoji/color picker you can already set per list actually mean something beyond the sidebar square.
