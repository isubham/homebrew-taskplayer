"Recent" is a pinned entry that lives above the "Your Lists" header — not a real list, so no color/emoji/rename, no drag-reorder, no "Add task". Clicking it opens a read-only page of the last 6 distinct tasks played, most-recent first, deduped (playing the same task again just moves it back to #1 instead of adding a duplicate row). Because tasks can come from any list, every row also shows which list it belongs to. All three page layouts below share the same sidebar entry — only the page itself changes.

The sidebar entry

Same .list-item row every list already uses, so it doesn't look bolted-on — just with a neutral clock glyph in a grey tile instead of a colored emoji square (since "Recent" has no color of its own), and no rename pencil on hover (there's nothing to rename).

2 · Card grid — Spotify's "Recently played" shelf

Closest to the metaphor the whole app is already leaning on. Six tiles instead of six table rows; each is its own little card with the list's color/emoji as the "cover art", the task name, its list, when it was last played, and a slim progress bar along the bottom edge. Reads more like a home-screen shelf than a data table — better suited to a smaller "Recent" shelf embedded elsewhere too, not just its own full page.

3 · Compact stacked rows — borrowed from the "Recent sessions" rail card

The Now Playing rail already has a small "Recent sessions" list in this exact shape; this is that same row style promoted to a full page, with a numbered rank badge standing in for the position (#1 = played most recently) instead of the sidebar's usual track number. Sits between the density of the table and the visual weight of the card grid.
